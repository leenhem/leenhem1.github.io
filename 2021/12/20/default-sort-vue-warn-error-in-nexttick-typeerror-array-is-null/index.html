default-sort [Vue warn]: Error in nextTick: "TypeError: array is null"

发表于2021-12-20|更新于2022-06-24|VUE网页开发
|阅读量:

介绍

在vue项目中使用element-ui时报错：[Vue warn]: Error in nextTick: “TypeError: array is null”
配置了:default-sort=”{prop: ‘internal_ip’, order: ‘ascending’}” 后就报错
报错信息如下：

[Vue warn]: Error in nextTick: “TypeError: array is null”

found in

—>
at packages/table/src/table.vue
at src/views/kubernetes/clusters.vue
at src/layout/components/AppMain.vue
at src/layout/index.vue
at src/App.vue
vue.runtime.esm.js:619
TypeError: array is null

报错分析

原因就是给el-table的数据源初始时设置为了null，而在使用了default-sort属性时，table在默认排序时对数据源进行map当然报错。

问题解决 default-sort [Vue warn]: Error in nextTick: “TypeError: array is null”

将list: null改为list: []就解决了，将数据源数组初始值改为[] 空数组即可。


再次访问，就不会报错了

类似的错误

[Vue warn]: Error in nextTick: “TypeError: Cannot read property ‘map‘ of null“
也是这样解决，一般原因就是我们在绑定数据时，绑定了null值
