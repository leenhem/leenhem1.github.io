Python 虚拟环境搭建之Virtualenv

发表于2021-08-16|更新于2022-06-24|Python
|阅读量:

前言

什么是虚拟环境？

从百度百科得知：以专利的实时动态程序行为修饰与模拟算法，直接利用本机的 OS，模拟出自带与本机相容 OS 的虚拟机（Vista 下可模拟 Vista、XP，Windows 7 下则可模拟 Windows 7、Vista、XP），也称为 “虚拟环境”。

功能：每一个环境都相当于一个新的 Python 环境。你可以在这个新的环境里安装库，运行代码等.。

为什么需要使用虚拟环境？

众所周知 Python 有强大的兼容性，粘附性强，同时却有些许库之间并不兼容。

什么时候需要使用虚拟环境？

需要使用某库，但是和已安装的库存在兼容性问题，导致运行报错。

某些库迭代更新，旧项目使用旧库，更新成新库会导致项目无法正常运行。

原理

各虚拟环境相当于不同的柜子，每个柜子存放物品不同都不会影响其他柜子，指定不同的虚拟环境完成不同的项目。

搭建前

请确保 Python 已安装至使用的电脑中 (最好已经配置好了环境变量）

请确保 pip 命令能够正常使用，且能正常安装库

安装

Linux安装

pip install virtualenv virtualenvwrapper -i https://pypi.doubanio.com/simple/

Windows安装

pip install virtualenvwrapper-win -i https://pypi.doubanio.com/simple/

环境配置

Linux

sed ‘$a export WORKON_HOME=~/Envs’ ~/.bashrc
source /usr/local/bin/virtualenvwrapper.sh
source ~/.bashrc

Windows

set PATH=%PATH%;WORKON_HOME=D:\Envs

常用命令

创建Python虚拟环境，命名为venv

Mac:code404.icu michael$ virtualenv –no-site-packages venv
Using base prefix ‘/usr/local/…/Python.framework/Versions/3.4’
New python executable in venv/bin/python3.4
Also creating executable in venv/bin/python
Installing setuptools, pip, wheel…done.

命令virtualenv就可以创建一个独立的Python运行环境，我们还加上了参数–no-site-packages，这样，已经安装到系统Python环境中的所有第三方包都不会复制过来，这样，我们就得到了一个不带任何第三方包的“干净”的Python运行环境。

进入虚拟环境

新建的Python环境被放到当前目录下的venv目录。有了venv这个Python环境，可以用source进入该环境

Mac:code404.icu michael$ source venv/bin/activate
(venv)Mac:code404.icu michael$

注意到命令提示符变了，有个(venv)前缀，表示当前环境是一个名为venv的Python环境。
在venv环境下，用pip安装的包都被安装到venv这个环境下，系统Python环境不受任何影响。

退出当前虚拟环境，使用deactivate命令

(venv)Mac:code404.icu michael$ deactivate
Mac:code404.icu michael$

此时就回到了正常的环境，现在pip或python均是在系统Python环境下执行。
